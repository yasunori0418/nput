# ADR-0031: 成功時デフォルト沈黙化・`-v` で配置レポート opt-in・`--debug` で nix コマンド開示・`--quiet` 廃止（ADR-0023 §2 改訂）

- ステータス: 採用
- 日付: 2026-06-23
- 関連: ADR-0007, ADR-0023, ADR-0024, ADR-0026, `docs/spec.md`, `docs/design.md`
- 改訂対象: ADR-0023 §2「出力ストリーム規律・冗長度フラグ」の冗長度方針を反転（既定 ON → 既定沈黙）。ストリーム規律（stdout 機械可読専有・warning/error は stderr）と終了コード表は不変。

## 背景

ADR-0023 §2 は MVP の冗長度フラグを **`--quiet`（進捗 / レポート抑制）/ `--verbose`（内部 nix コマンド開示）** とし、**成功時の配置レポートを既定で stderr に出す**設計だった。実装後、次の不都合が明確になった。

1. **UNIX 哲学「沈黙は金」に反する**。成功は最頻ケースで、毎回 `nput: apply <name> 完了 ...` と per-target 行が stderr に出る。`shellHook`（`nput apply --all --project-root --no-wait`）から入室のたびにレポートが流れ、ノイズになる。
2. **`--quiet` が事実上の既定運用になる**。静音が欲しい呼び出し側（shellHook・スクリプト・CI）は一律 `--quiet` を付ける必要があり、「既定で出して個別に黙らせる」構図が逆。
3. **`--verbose` の意味が「nix コマンド開示」に閉じていて、本来欲しい「もっと詳しい配置結果」と噛み合わない**。冗長度の軸（成功レポートの詳しさ）とデバッグの軸（内部コマンド開示）が 1 フラグに混載していた。

## 決定

### 1. 成功時はデフォルト沈黙（warning / error を除き stderr 無出力）

`apply` / `reset` / `rollback` の**成功時配置レポート（サマリ + per-target 行）・try-lock skip 通知・`apply --all` 完了サマリ・「対象 config なし」通知**を、既定では出さない。成功は終了コード 0 が語る（沈黙は金）。

- **warning（foreign symlink 等・ADR-0015）と error は常時 stderr に出す**（沈黙対象外）。
- **`reset` の確認プロンプト（削除対象表示 + y/N）は存続**する（破壊的操作の同意・ADR-0021）。`reset` 中止通知も存続する。
- **stdout の機械可読出力（`gitignore` の列挙・`apply --dryrun` / `reset --dryrun` のプラン・`list-generations`）は一切沈黙化しない**（ストリーム規律不変・ADR-0023 §2）。

### 2. `-v` / `--verbose` を「配置レポート opt-in」に再定義

`-v` / `--verbose` を付けたときだけ、配置レポート（サマリ + per-target 行）・skip 通知・`--all` 完了サマリを stderr に出す。ADR-0023 の既定 ON レポートを、そのまま `-v` 下の出力に移す（出力内容は不変・既定の有無だけ反転）。

### 3. 内部 nix コマンド開示を `--debug` に分離

ADR-0023 で `--verbose` が担っていた「内部実行する `nix` コマンドの逐次開示（`nput: + nix eval ...`）」を **`--debug`** に移す。冗長度（配置結果の詳しさ＝`-v`）とデバッグ（内部コマンド可視化＝`--debug`）を直交させる。`--help` / `rootCmdLong` の透明性開示文も `--debug` を案内する。

### 4. `--quiet` を廃止

既定が沈黙になったため `--quiet` は不要（抑制すべき既定出力が無い）。フラグを削除する。静音は既定、詳細は `-v`、内部コマンドは `--debug` の 3 状態で足りる。

## 根拠

- **沈黙は金**: 成功の最頻ケースを無音にし、失敗・警告だけが目に入る。shellHook 既定（`--no-wait`）が `--quiet` 無しで静かになる。
- **opt-in レポート**: レポートを見たい対話運用は `-v` を足すだけ。出力内容は ADR-0023 と同一なので実装は gating の反転（`!flagQuiet` → `flagVerbose`）で済む。
- **軸の分離**: 「結果の詳しさ」と「内部コマンドのデバッグ」は別目的。`-v`（人向けレポート）/ `--debug`（troubleshooting）に分けると、片方だけ欲しい運用に素直に応える。
- **`--quiet` 廃止**: 既定沈黙下で `--quiet` は no-op にしかならず、残すと「既定で出る」という誤解を招く。破壊的フラグ削除（`!`）だが MVP 段階・利用者は限定的で許容。

## 影響

- **`cmd/nput/`**: `main.go`（`--quiet` 削除・`-v`/`--verbose` 再定義・`--debug` 追加・`rootCmdLong` 開示文）、`apply.go` / `reset.go` / `rollback.go`（レポート gating を `flagVerbose` に反転）、`nix.go`（nix コマンド開示を `flagDebug` gating）、`init.go`（同）。
- **`docs/spec.md`**: グローバルフラグ表（`--quiet` 削除・`-v`/`--verbose` 再定義・`--debug` 追加）、「出力ストリームと終了コード」節（既定沈黙・`-v` opt-in・`--debug` を明記）。ストリーム規律・終了コード表は不変。
- **`docs/design.md`**: 出力・終了コード規約の 1 行を既定沈黙・`-v`/`--debug` に更新。
- **本 ADR は出力文字列の英語化（#53）・コメント英語化（#54）を含まない**（日本語のまま gating だけ変える）。

## 棄却した代替案

- **`--quiet` を残し既定 ON のまま**: ADR-0023 の現状。沈黙は金に反し shellHook ノイズが残る。
- **`-v` に nix コマンド開示も残す（`--debug` を作らない）**: 軸が混載し、レポートだけ欲しい運用が内部コマンドまで浴びる。
- **`-v` を多段（`-vv` で nix コマンド）にする**: cobra で count フラグは可能だが、目的が異なる 2 軸を 1 フラグの段数に畳むと意味が不明瞭。独立フラグの方が明快。
